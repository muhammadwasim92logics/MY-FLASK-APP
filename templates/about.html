User Manual for the Database Management Application


Introduction

This web application allows users to securely manage a database of user records, including email addresses and passwords. The application provides features to add, view, update, and delete records, with additional security for accessing the database.

Getting Started:

  • Launch the Application: Open your web browser and navigate to the application using the provided link or http://localhost:8800 if running locally.
  • Homepage: The homepage provides a simple form to add new user records to the database.

Features and Instructions

1. Add a New User

  1. Enter an email address and password in the form on the homepage.
  2. Click the "Submit" button to save the new user record.

Result: The user record will be saved in the database.

2. View the Database

Accessing the Database:

  1. Navigate to the Database Access Page.
  2. Enter the secure access key provided by the administrator when prompted.
  3. If the key is correct, the database records will be displayed.

Note:

  • You are allowed a maximum of 3 attempts to enter the correct access key.
  • After 3 incorrect attempts, access to the database will be temporarily blocked.

3. Update a User Record

  1. On the database view page, find the user you want to update.
  2. Click the "Update" button next to the user record.
  3. Enter the updated email address and password in the form.
  4. Click "Submit" to save the changes.

Result: The selected user record will be updated in the database.

4. Delete a User Record

  1. On the database view page, find the user you want to delete.
  2. Click the "Delete" button next to the user record.

Result: The selected user record will be permanently removed from the database.

5. Security Features

  • Secure Access: The database is protected by a secure access key. The access key must be entered correctly to view the database records.
  • Failed Attempts: After 3 incorrect access attempts, the system will temporarily block further access to the database. Wait for a while before trying again.

6. Flash Messages

Purpose: Flash messages appear at the top of the screen to notify users about actions and errors.

  • "Record added successfully!" after adding a new user.
  • "Incorrect key! 2 attempts remaining." after entering the wrong access key.

Troubleshooting

  • Forgot Access Key: Contact the administrator to retrieve or reset the access key.
  • Cannot Update/Delete: Ensure the record exists. If it doesn't, verify using the database view page.
  • Session Errors: If you encounter issues like being logged out unexpectedly, clear your browser cookies and try again.
